## The short version

I'm a data person who drifted into machine learning and never looked back. I like problems that don't have obvious answers — the kind where you have to dig through noisy data, question your assumptions, and build something that actually holds up in the real world.

My work lives somewhere between analysis and engineering: I care about *what* the model says, *why* it says it, and whether it'll still make sense six months from now in production.
